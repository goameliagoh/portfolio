EduHelper Website

EduHelper is an educational website I developed with three teammates, utilizing Node.js, Express, and an SQL database with the MVC architecture. I took charge of the comments and reviews features, where I implemented CRUD and also input validation using JOI and user authorization with Json Web Token (JWT). This setup allows only logged-in users to create comments and reviews, and ensures that only the original poster can edit or delete their submissions. The first image shows the index page users see when they first visit the site. The second image displays how reviews would be displayed for each lecturer videos. Currently, the user logged in is "John Doe." The next two images highlights the pop-ups for creating and editing reviews, designed using Gignite.AI. The fifth image then showcases how the comment section would look like for each discussions. In this image, you can also see how I integrated Google Cloud's translation API to translate non-English comments to English and also display the detected language. The next two images shows the pop-ups for creating and editing comments, also designed with Gignite.AI. Finally, the last image presents the user dashboard, where users can view, edit, or delete their reviews and see the total number of reviews they have posted.

Ngee Ann City Game

Ngee Ann City is a strategic building game where players construct and manage various types of buildings such as commercial and industrial buildings to earn points, scores, and also coins. This game features two modes: Arcade Mode and Free Play Mode, each with its own unique scoring system. This game project was successfully created by our team of 5, which consisted of three developers, including myself, alongside a dedicated Scrum Master and Product Owner, both of whom also contributed as developers. Our lecturer acted as the stakeholder for this project. Our team made use of HTML, CSS, JS for this project and adopted the SCRUM framework to manage the project, completing it over four two-week sprints. We utilized JIRA for tracking our sprints, conducted daily scrums and sprint reviews via Microsoft Teams, and performed sprint retrospectives using Retrotool.io. The first five images illustrate our game’s design, showcasing the different game modes, saved games, and also the high score lists. Following that, the next few images gives you an insight into our development process, showcasing our sprint issues, retrospectives, and burndown charts for each of the four sprints.

Heart2Heart Chatbot

Heart2Heart is a therapy chatbot I developed with two teammates using IBM Watsonx Assistant. It allows users to book, reschedule, and cancel appointments with their chosen therapist, and provides a space to vent and express feelings. I implemented sentiment analysis to ensure responses are empathetic and relevant, and made it so the chatbot can recommend resources like YouTube meditation videos or even emergency resources based on user emotions. The chatbot also includes a FAQ section for quick access to common questions. I was in charge of developing the venting system. The first image provides an overview of the dialogue flows I created for Heart2Heart's venting system. The subsequent images offer a rough demonstration of how the chatbot would handle conversations with users facing challenges in school or work. Additionally, a table of action, intent, entities and utterances is also shown for each respective conversation. These images highlight just a portion of the system’s capabilities, focusing on this specific scenario.
